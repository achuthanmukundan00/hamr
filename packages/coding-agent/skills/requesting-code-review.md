# Requesting Code Review

## Core Rule

Review early, review often.

## Workflow

1. Review the diff against the plan.
2. Ask for a reviewer that is scoped to the actual changes.
3. Request feedback before merge or commit when the change is non-trivial.
4. Re-run verification after fixes.

## Guardrails

- Keep the review focused on the changed files.
- Ask for issues first, not praise.

# Brainstorming

## Core Rule

Do not write code until the goal, constraints, and success criteria are clear.

## Workflow

1. Read the current project context.
2. Ask one question at a time.
3. Offer 2-3 approaches with tradeoffs and a recommendation.
4. Present a short design and wait for approval.

## Guardrails

- Split large ideas into sub-projects.
- Keep the design small and testable.
- If visuals help, show a visual instead of describing it.

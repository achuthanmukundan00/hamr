# CodeRabbit Review

## Core Rule

Review the local diff as if it were a PR.

## Workflow

1. Gather the unstaged and staged diffs.
2. Read the changed files with surrounding context.
3. Report strengths, then issues, then a verdict.
4. Rank issues by severity and include file:line references.
5. Offer minimal fixes only when you are confident.

## Output

- Summary
- Walkthrough
- Strengths
- Issues
- Recommendations
- Verdict

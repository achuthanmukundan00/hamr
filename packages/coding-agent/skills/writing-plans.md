# Writing Plans

## Core Rule

Write the plan as if the next engineer knows nothing and will skip the obvious parts.

## Plan Shape

1. Map the files and responsibilities first.
2. Break the work into 2-5 minute steps.
3. Give exact paths, commands, and verification steps.
4. Keep each step independently shippable.
5. Self-review for placeholders, contradictions, and missing tests.

## Output

- Prefer small, focused tasks.
- Include the test command for every behavior change.
- Save the plan in the repo's plan docs when the project expects that.

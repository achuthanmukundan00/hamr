# Writing Skills

## Core Rule

Treat skills like behavior-shaping code.

## Workflow

1. Start with the trigger, not the process.
2. Keep the description specific and short.
3. Write the smallest instruction set that changes behavior.
4. Remove redundant prose and loopholes.
5. Test the skill against real failure cases.

## Guardrails

- Optimize for discovery and compliance, not literary quality.
- Keep high-frequency skills especially short.

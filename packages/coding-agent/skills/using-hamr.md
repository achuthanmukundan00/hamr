# Using Hamr

## Core Rule

If a bundled skill or prompt template clearly fits the task, use it before improvising.

## What To Do

1. Read the current project context first.
2. Check the bundled skills and prompt templates for a direct match.
3. Prefer the smallest workflow that still gives a correct result.
4. Ask one clear question at a time when requirements are unclear.
5. Keep work verifiable and file-scoped.

## Defaults

- Design before code when the request is ambiguous or creative.
- Verify before declaring completion.
- Use worktrees for isolated implementation.
- Review diffs before merging.

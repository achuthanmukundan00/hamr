# Receiving Code Review

## Core Rule

Verify the feedback before acting on it.

## Workflow

1. Read the comment in full.
2. Check the code and the surrounding context.
3. Clarify anything ambiguous before changing it.
4. Apply only the changes that are technically justified.
5. Re-verify after the fix.

## Guardrails

- Do not implement vague feedback blindly.
- Technical correctness beats performative agreement.

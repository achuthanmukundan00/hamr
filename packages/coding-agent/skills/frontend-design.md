# Frontend Design

## Core Rule

Choose a deliberate visual direction and execute it consistently.

## Workflow

1. Start from the user's purpose and audience.
2. Define typography, color, layout, and motion as a single system.
3. Avoid default, interchangeable UI patterns.
4. Keep the page responsive and usable on mobile.

## Guardrails

- Do not ship bland, template-like interfaces.
- Preserve the repo's existing design language when one already exists.
